Task 5

This task is about displaying advertising banners on web sites. Its needed to show a proper adv, impression act. The success criterion is maximized CTR (clickthrough rate) when the number of the placeholders are limited, e.g., vacant banners in web sites. Could you please design and create a recommender system that could help in this case? Please, could you reflect on the response time, since it could be an important requirement. Historical data for several days is posted here[1].

We propose to use Vowpal Wabbit[2]. There are few realizations: PyPpark package[3], bare python library, console application, and API. In case of lack of computing resources, use sampling. Exploratory data analysis is the assessed part of the assignment.

Success criteria A method for estimating the show cost is proposed Demonstrated the benefits of using Behavior insights presented; client segments proposed The solution works in high loaded and parallel mode

[1] https://www.kaggle.com/c/avazu-ctr-prediction/data

[2] https://vowpalwabbit.org/

[3] https://github.com/Azure/mmlspark/blob/master/docs/vw.md

class NegativeSampler:
    """
    Class for negative samples creation for Support portal requests dataset

    Parameters
    ----------
    month_threshold (int): requests for NS are taken from this time threshold
    ns_amount (int): amount of new negative requests, which weren't made by user with user_id
    item_storage (numpy.ndarray): all unique values of TARGET column
    Sampling steps (methods):
    ---------------
        - _set_bin_target
        - _append_neg_samples
        - _fit_df
        - _transform_df
    """

    def __init__(self,
                 day_threshold: int = 1,
                 ns_amount: int = 10):
        
        self.day_threshold = day_threshold
        self.ns_amount = ns_amount
        self.item_storage = None

    def _set_bin_target(self, X: pd.DataFrame) -> pd.DataFrame:
        """
        Set binary target (only 1).
        Parameters
        ----------
        X (pd.DataFrame): Initial dataset after preprocessing step

        Returns
        --------
        X_ (pd.DataFrame): Initial dataset with one new column "y", as binary target
        """
        X_ = X.copy(deep=True)
        X_['hour'] = pd.to_datetime(X_['hour'])
        X_['hour'] = pd.to_datetime(X_['hour'].dt.date)
        X_['weekday'] = X_['hour'].dt.weekday
        X_['weekday'] = X_['weekday'].replace({0: 'Mon', 1: 'Tues', 2: 'Wed', 3: 'Thurs', 4: 'Fri', 5: 'Sat', 6: 'Sun'})
        return X_

    def _generate_neg_samples(self, user_rows: pd.DataFrame):
        """
        Create new negative sample rows with new request and 'y' equals to 0.

        Parameters
        ----------
        user_rows (DataFrame): Samples(rows) of one user from initial dataset
        mode (str) : {'train', 'test'}, default='train'
                    Samples are generated for:
                - 'train' dataset, size of new generated rows for one user is equal to "self.ns_amount".
                - 'test' dataset, size of new generated rows for one user is equal to all possible S.TARGETs.

        Returns ns_df (DataFrame): dataset with only negative samples of one user
        -------
        """

       
        uu_req = set(user_rows[TARGET])
        not_uu_reqs = list(set(self.item_storage) - uu_req)

        if len(not_uu_reqs) >= self.ns_amount:
            generated_requests = random.sample(not_uu_reqs, self.ns_amount)
        else:
            generated_requests = random.sample(not_uu_reqs, len(not_uu_reqs))
   

        # get row-id of last user request
        user_rows = user_rows.sort_values(by='hour')
        user_row_id = user_rows.index[-1]

        # get one row by row-id for user-data extraction
        user_row = user_rows.loc[user_row_id]

        # get date of last user request
        last_req_date = user_rows['hour'].loc[user_row_id]
        last_req_weekday = user_rows['weekday'].loc[user_row_id]

        negative_samples = []
        for req in generated_requests:
            new_row = {}
            for col in user_rows.columns:
                if col == 'hour':
                    new_row[col] = last_req_date
                elif col == 'weekday':
                    new_row[col] = last_req_weekday
                elif col == TARGET:
                    new_row[col] = 1
                else:
                    new_row[col] = user_row[col]

            negative_samples.append(new_row)

        ns_df = pd.DataFrame(negative_samples)

        return ns_df

    def fit(self, X: pd.DataFrame, y: Optional[pd.DataFrame] = None):
        pass

    def transform(self, X: pd.DataFrame) -> pd.DataFrame:
        """
        Create new negative sample rows for each user.
        -------

        Parameters:

        X (DataFrame): Original dataset
        -------

        Returns:

        Returns negative_samples_df (DataFrame): dataset with only negative samples of all users
        """

        X = self._set_bin_target(X)
        date_boarder = max(X['hour']) - relativedelta(days=self.day_threshold)
        self.item_storage = X[X['hour'] <= date_boarder]['id'].unique()

        negative_samples_df = X.groupby('device_id').apply(self._generate_neg_samples)
        new_df = X.append(negative_samples_df, ignore_index=True, sort=False)
        return new_df

Conclusion

  • Some categories seem to have a very large number of unique values, which suggests that the values may be specific to a small number of training examples and may be candidates for deletion to reduce the number of functions, since they do not include generalizing information.

  • In particular, "device_id" and "device_ip" grow linearly with the number of examples, so we decided to remove them from our functions when training the model.

  • From the point of view of click distribution, sampling data from 700k rows is not very informative for the entire dataset; however, there seem to be some interesting features, for example, we can observe that in category C1, enter "0", although there are not many examples available, apparently constantly lead to a click event; this suggests that the function C1-0 can be a good indicator of clicks.
